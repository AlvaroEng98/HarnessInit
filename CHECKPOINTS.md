# CHECKPOINTS

Criterios objetivos usados por el `reviewer_agent` para evaluar una feature.

- C1 — Código compila/interpreta sin errores
- C2 — `init.sh` pasa verde
- C3 — Todos los tests pasan
- C4 — Trazabilidad R<n> ↔ tests completa
- C5 — `tasks.md` completamente marcado `[x]`
- C6 — Sin TODOs, debug prints, ni código comentado
- C7 — La feature respeta `docs/architecture.md` y `docs/conventions.md`
